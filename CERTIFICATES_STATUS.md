# حالة الشهادات والأسعار

## المشكلة
الصفحة الرئيسية تظهر "لا توجد شهادات متاحة حالياً" بدلاً من الشهادات الأربع مع الأسعار.

## السبب
الشهادات موجودة في ملف `prisma/seed.ts` لكن لم يتم إدراجها في قاعدة البيانات بعد.

## الحل

### اختر أحد الطريقتين:

#### الطريقة 1: إدراج سريع (الأفضل)
```bash
npm run db:insert-certs
```

**المخرجات المتوقعة:**
```
🔄 بدء إدراج الشهادات...

✅ تم إنشاء: مبتدئ لغة المرجع - $49
✅ تم إنشاء: مطور لغة المرجع - $99
✅ تم إنشاء: خبير لغة المرجع - $149
✅ تم إنشاء: محترف لغة المرجع المعتمد - $199

✨ اكتمل إدراج الشهادات بنجاح!
```

#### الطريقة 2: إعادة تعيين كاملة
```bash
npx prisma migrate reset
npm run db:seed
```

## معلومات الشهادات الأربع

### 1️⃣ مبتدئ لغة المرجع
- **السعر:** $49 (السعر الأصلي: $99)
- **المستوى:** مبتدئ
- **المدة:** أسبوعين
- **عدد الأسئلة:** 20
- **مدة الاختبار:** 30 دقيقة
- **درجة النجاح:** 60%

- **عدد الأسئلة:** 30

- **درجة النجاح:** 70%

### 3️⃣ خبير لغة المرجع
- **السعر:** $149 (السعر الأصلي: $299)
- **المستوى:** متقدم
- **المدة:** 6 أسابيع
- **عدد الأسئلة:** 40

## التحقق من النجاح

### 1. تأكد من إدراج البيانات:
```bash
node scripts/check-certificates.js
```

### 2. ابدأ الخادم:
```bash
npm run dev
```

### 3. افتح المتصفح:
```
http://localhost:3000
```

### 4. اذهب إلى قسم الشهادات
يجب أن ترى:
- الشهادات الأربع بشكل واضح
- أسعار مبارزة لكل واحدة ($49, $99, $149, $199)
- نسب الخصم (50%, 50%, 50%, 50%)
- زر "ابدأ الاختبار الآن" لكل شهادة

## الملفات المرتبطة

- 📄 `/scripts/insert-certificates.js` - Script إدراج البيانات
- 📄 `/scripts/check-certificates.js` - التحقق من البيانات
- 📄 `/INSERT_CERTIFICATES.md` - دليل مفصل
- 📄 `/prisma/schema.prisma` - نموذج قاعدة البيانات
- 📄 `/prisma/seed.ts` - بيانات الشهادات الأصلية
